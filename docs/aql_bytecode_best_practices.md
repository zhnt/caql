# AQL字节码编写最佳实践

## 📋 **指令使用规范**

### ✅ **正确模式**

#### **常量使用**
```assembly
# ❌ 错误：直接使用常量
ADD R1, R0, K0

# ✅ 正确：先LOADK再ADD
LOADK R3, K0
ADD   R1, R0, R3
MMBIN R1, R0, 6    # 对结果寄存器进行元方法绑定
```

#### **条件跳转**
```assembly
# ✅ 正确：LEI + JMP模式
LEI   R0, 0, 0     # n <= 0，如果为真继续，为假跳转
JMP   2            # 跳转偏移（不是两个参数）
```

#### **MMBIN使用**
```assembly
# ✅ 正确：
ADD   R2, R0, R1    # R2 = R0 + R1 (算术指令)
MMBIN R0, R1, 6     # 如果ADD失败，调用__add(R0,R1)，结果自动存到R2

# ❌ 错误：对结果寄存器进行绑定
ADD   R2, R0, R1
MMBIN R2, R0, 6     # 错误！应该是操作数R0,R1
```

**关键点**：
- MMBIN的A,B参数是**操作数寄存器**（如R0,R1）
- 结果自动存储到**前一条算术指令的目标寄存器**（如R2）
- 这与Lua 5.4.8完全一致

#### **函数调用参数传递**
```assembly
# ✅ 正确：参数必须在连续寄存器中
GETTABUP R3, 0, K0    # 获取函数
LOADK    R4, K1       # 参数1 -> R4 (函数后的第一个寄存器)
CALL     R3, 2, 2     # 函数在R3，2=函数+1个参数，2=期望1个返回值
```

### 🚫 **常见错误模式**

1. **直接使用常量进行运算**
2. **MMBIN参数错误**
3. **JMP使用两个参数**
4. **函数调用参数位置错误**

### 📝 **验证清单**

- [ ] 所有ADD指令都使用寄存器与寄存器
- [ ] 所有常量都先LOADK到寄存器
- [ ] MMBIN第一个参数是结果寄存器
- [ ] JMP只有一个跳转偏移参数
- [ ] 函数调用参数在连续寄存器中
